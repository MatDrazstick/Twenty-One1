===========================================
BUG 1: Hidden Card Not Displayed to Player
===========================================

PROBLEM:
--------
The game was not showing the player their own hidden (face-down) card during their turn. 
This made it difficult for the player to make informed decisions about whether to draw or stay.

ROOT CAUSE:
-----------
The interactiveGame.ts file was displaying the player's hand and visible score, but never 
showed the hidden card information to the current player.

FIX APPLIED:
------------
Location: /workspaces/Twenty-One/src/interactiveGame.ts (Lines ~138-143)

Added code to display the hidden card to the current human player during their turn:

```typescript
// Bug 1 Fix: Show hidden card to current player only
if (currentPlayer.faceDownCard) {
  console.log(`Your hidden card: ${currentPlayer.faceDownCard.toString()}`);
}
```

TESTING:
--------
- Created test in allBugFixTests.ts (Bug 1 Test section)
- Verified that faceDownCard is accessible and can be displayed
- Confirmed card information is shown only to the current player

===========================================
BUG 2: Timer Force Draw Doesn't Continue
===========================================

PROBLEM:
--------
When the timer ran out and forced a player to draw, the game would not automatically 
proceed to the AI's turn. The game would wait for another user action before the AI 
could take its turn, causing the game flow to stall.

ROOT CAUSE:
-----------
The timer callback in Game.ts was calling playerDraws() but the async nature wasn't 
properly managed, and the game loop in interactiveGame.ts wasn't continuing after 
a forced draw.

FIX APPLIED:
------------
Location: /workspaces/Twenty-One/src/Game.ts (Lines ~494-510)

Improved the timer expiration logic to properly handle async forced draws:

```typescript
// Time's up - force action
if (this.turnTimeRemaining <= 0) {
  this.stopTurnTimer();
  
  const currentPlayer = this.players[this.currentPlayerIndex];
  const currentPlayerStayed = (this.currentPlayerIndex === 0) ? 
    this.player1Stayed : this.player2Stayed;
  
  // Bug 3 Fix: Check if player has already busted or stayed
  if (currentPlayer.isBusted || currentPlayerStayed) {
    console.log(`Time's up! Player has already ${currentPlayer.isBusted ? 'busted' : 'stayed'}.`);
    // Just switch turn, don't force draw
    this.switchTurn();
  } else {
    // Bug 2 Fix: Force the player to draw and let it complete
    console.log(`Time's up! Forcing draw...`);
    this.playerDraws().catch(err => console.error('Error forcing draw:', err));
  }
}
```

The key improvement is that playerDraws() naturally switches turns when it completes,
allowing the game loop to continue with the AI's turn.
