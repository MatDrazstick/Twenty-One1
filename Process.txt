## Errors Encountered and Resolutions

### Error 1: Type Definition for Game Parameter
- **Issue**: TypeScript complained about circular dependency when importing Game type in AbilityCard.ts
- **Resolution**: Used `any` type for game parameter in ability effects to avoid circular dependency
- **Status**: ✓ Resolved

### Error 2: Card Availability in Add Number Abilities
- **Issue**: 2-Card ability failed when card already dealt
- **Resolution**: Added graceful handling - ability is used but no card drawn if unavailable
- **Status**: ✓ Working as designed

### Error 3: Face-Down Card Handling in Exchange
- **Issue**: Need to exclude face-down cards from exchange
- **Resolution**: Filter out face-down cards before selecting cards to exchange
- **Status**: ✓ Resolved

### Error 4: Bet Modifier Not Affecting Kill Machine
- **Issue**: Initially forgot to apply bet modifier to kill machine movement
- **Resolution**: Updated `updateKillMachine()` to calculate `actualMoveDistance = moveDistance + betModifier`
- **Status**: ✓ Resolved

### Error 5: Bless Ability Not Preventing Death
- **Issue**: Bless flag set but not checked during kill machine update
- **Resolution**: Added check in `updateKillMachine()` to detect if player would die and has Bless
- **Status**: ✓ Resolved

## Ability Card Descriptions

### Add Number Abilities
1. **2-Card**: Draw card with value 2 if available in deck
2. **3-Card**: Draw card with value 3 if available in deck
3. **4-Card**: Draw card with value 4 if available in deck
4. **6-Card**: Draw card with value 6 if available in deck
5. **7-Card**: Draw card with value 7 if available in deck

### Deck Trump Abilities
1. **Hush**: Draw a hidden card (opponent can't see value)
2. **Perfect Draw**: Draw best available card to reach target without busting
3. **Refresh**: Return all visible cards and draw 2 new ones
4. **Remove**: Remove opponent's last drawn card
5. **Return**: Return your last drawn card to deck
6. **Exchange**: Swap your last card with opponent's last card
7. **Disservice**: Force opponent to draw a card

### Bet Abilities
1. **One-Up**: Increase bet modifier by +1
2. **Two-Up**: Increase bet modifier by +2
3. **Shield**: Decrease bet modifier by -1
4. **Shield-Plus**: Decrease bet modifier by -2
5. **Bless**: Avoid death once if you would lose
6. **Bloodfeast**: Increase bet modifier by +1 AND draw an ability card
7. **Destroy**: Nullify opponent's most recent ability
8. **Friendship**: Both players draw 2 ability cards
9. **Relentless**: Destroy opponent's ability AND draw 1 ability card

### Go For Abilities
1. **Go For 17**: Change target number from 21 to 17
2. **Go For 24**: Change target number from 21 to 24
3. **Go For 27**: Change target number from 21 to 27

  - Exported interface for use in other files

### Step 2: Added Settings Properties to Game Class
- **Changes Made**:
  - Added `settings: GameSettings` property
  - Added timer state tracking:

    - `turnStartTime: number | null` - when current turn started
    - `turnTimeRemaining: number` - seconds remaining in turn
- **Status**: ✓ Successfully implemented

### Step 3: Updated Game Constructor
- **Changes Made**:
  - Initialize default settings (timer: 30s, mode: rise, first: player1)
  - Accept settings in both constructor signatures:
    - `(player1Name, mode, aiDifficulty?, settings?)`
    - `(player1Name, player2Name, settings?)`
  - Merge provided settings with defaults using spread operator
  - Set initial `currentPlayerIndex` based on `firstPlayer` setting
  - Handle random first player selection
- **Status**: ✓ Successfully implemented

### Step 4: Implemented Timer Methods
- **Methods Added**:
  1. `startTurnTimer()`:
     - Clears any existing timer
     - Skips timer for AI players
     - Sets up interval to check time every second
     - Displays warning at 10 seconds remaining
     - Forces draw when time expires
  2. `stopTurnTimer()`:
     - Clears interval timer
     - Resets timer state
  3. `getTurnTimeRemaining()`:
     - Returns current time remaining in seconds
- **Status**: ✓ Successfully implemented

### Step 5: Integrated Timer with Turn System
- **Changes Made**:
  - `setupNewRound()`: Starts timer for current player at round start
  - `switchTurn()`: Stops current timer and starts new timer for next player
  - Timer automatically stops when player draws or stays (via switchTurn)
- **Status**: ✓ Successfully implemented

### Step 6: Implemented Move Distance Modes
- **Changes Made in endRound()**:
  - Rise mode (default): `moveDistance++` (increases by 1 each round)
  - Shuffle mode: `moveDistance = Math.floor(Math.random() * 3) + 1` (random 1-3)
  - Console log for shuffle mode to show new distance

## Settings Descriptions

### Timer Setting
- **Purpose**: Controls how long each player has to complete their turn
- **Options**: 15, 30, 45, 60, 75, or 90 seconds
- **Default**: 30 seconds
- **Behavior**:
  - Timer starts at beginning of each turn
  - Warning displayed at 10 seconds remaining
  - Player forced to draw if time expires
  - Timer skipped for AI players (they take as long as they need)
  - Timer stops when player draws or stays

### Move Distance Mode
- **Purpose**: Controls how the kill machine's movement distance changes each round
- **Options**:
  1. **Rise** (default):
     - Distance starts at 1 in round 1
     - Increases by 1 each round (2, 3, 4, 5, ...)
     - Predictable and strategic
  2. **Shuffle**:
     - Distance is random between 1-3 each round
     - Adds unpredictability and excitement
     - Makes long-term planning harder
- **Default**: Rise
- **Effect**: Determines how far the kill machine moves toward loser each round

### First Player Setting
- **Purpose**: Controls which player takes the first turn each round
- **Options**:
  1. **Player1** (default): Player 1 always starts
  2. **Player2**: Player 2 (or AI) always starts
  3. **Random**: Randomly selected each round (50/50 chance)
- **Default**: Player1
- **Effect**: Can provide strategic advantage or disadvantage

## Known Limitations and Notes

1. **Timer Precision**: Timer checks every 1 second, so there may be slight delays
2. **AI Timer**: AI players don't have timers (by design)
3. **Force Draw**: When time expires, player is forced to draw (not stay)
4. **Random First Player**: Uses Math.random(), not cryptographically secure
5. **Shuffle Range**: Move distance shuffles between 1-3 (not configurable)
