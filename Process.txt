# Ability Card System Implementation Process

## Overview
This document tracks the implementation of the Ability Card system for the Twenty-One game, including all tests, errors encountered, and resolutions.

---

## Implementation Steps

### Step 1: Created AbilityCard.ts
- **Purpose**: Define the core ability card structure and all 24 ability cards
- **Categories Implemented**:
  - Add Number Abilities: 2-Card, 3-Card, 4-Card, 6-Card, 7-Card (5 cards)
  - Deck Trump Abilities: Hush, Perfect Draw, Refresh, Remove, Return, Exchange, Disservice (7 cards)
  - Bet Abilities: One-Up, Two-Up, Shield, Shield-Plus, Bless, Bloodfeast, Destroy, Friendship, Relentless (9 cards)
  - Go For Abilities: Go For 17, Go For 24, Go For 27 (3 cards)
- **Total**: 24 unique ability cards
- **Status**: ✓ Successfully implemented

### Step 2: Created AbilityDeck.ts
- **Purpose**: Manage the deck of ability cards
- **Features**:
  - Shuffle ability cards
  - Deal ability cards to players
  - Reset deck for new rounds
  - Track remaining cards
- **Status**: ✓ Successfully implemented

### Step 3: Extended Players.ts
- **Changes Made**:
  - Added `abilityHand: AbilityCard[]` property
  - Added `hasBless: boolean` property for Bless ability tracking
  - Added `printAbilityHand()` method to display abilities
  - Added `useAbility()` method to activate abilities
  - Updated `resetForNewRound()` to clear ability hand and bless status
- **Status**: ✓ Successfully implemented

### Step 4: Extended Game.ts
- **Changes Made**:
  - Added `abilityDeck: AbilityDeck` property
  - Added `targetNumber: number` property (default: 21)
  - Added `betModifier: number` property (default: 0)
  - Added `lastAbilityPlayed` tracking for Destroy/Relentless abilities
  - Updated `setupNewRound()` to:
    - Reset and shuffle ability deck
    - Deal 2 ability cards to each player
    - Reset game state (target number, bet modifier)
  - Updated `updateKillMachine()` to:
    - Apply bet modifier to move distance
    - Check for Bless ability to prevent death
  - Updated bust check to use dynamic `targetNumber` instead of hardcoded 21
  - Updated winner determination to use dynamic `targetNumber`
- **Status**: ✓ Successfully implemented

### Step 5: Extended AIPlayer.ts
- **Changes Made**:
  - Added `chooseAbility()` method with difficulty-based strategies:
    - **Level 1**: Random selection (30% chance to use any ability)
    - **Level 2**: Basic strategy (use simple beneficial abilities)
    - **Level 3**: Conservative strategy (prioritize defensive abilities)
    - **Level 4**: Smart strategy (tactical ability usage based on game state)
    - **Level 5**: Advanced strategy (optimal ability usage with complex decision-making)
  - Implemented private methods for each difficulty level:
    - `basicAbilityStrategy()`
    - `conservativeAbilityStrategy()`
    - `smartAbilityStrategy()`
    - `advancedAbilityStrategy()`
- **Status**: ✓ Successfully implemented

### Step 6: Created abilityTest.ts
- **Purpose**: Comprehensive testing of all ability cards and system integration
- **Tests Implemented**: 27 tests total
  1. Create and verify all ability cards
  2. AbilityDeck functionality
  3-9. Test individual Deck Trump abilities (2-Card, Perfect Draw, Hush, Remove, Return, Exchange, Disservice)
  10. Test Refresh ability
  11-15. Test Bet abilities (One-Up, Two-Up, Shield, Shield-Plus, Bless)
  16-19. Test complex Bet abilities (Destroy, Bloodfeast, Friendship, Relentless)
  20-22. Test Go For abilities (17, 24, 27)
  23. Test ability stacking
  24. Test Player.useAbility method
  25. Test AI Level 1 ability choice
  26. Test AI Level 5 ability choice
  27. Test full game integration
- **Status**: ✓ All tests passed

## Errors Encountered and Resolutions

### Error 1: Type Definition for Game Parameter
- **Issue**: TypeScript complained about circular dependency when importing Game type in AbilityCard.ts
- **Resolution**: Used `any` type for game parameter in ability effects to avoid circular dependency
- **Status**: ✓ Resolved

### Error 2: Card Availability in Add Number Abilities
- **Issue**: 2-Card ability failed when card already dealt
- **Resolution**: Added graceful handling - ability is used but no card drawn if unavailable
- **Status**: ✓ Working as designed

### Error 3: Face-Down Card Handling in Exchange
- **Issue**: Need to exclude face-down cards from exchange
- **Resolution**: Filter out face-down cards before selecting cards to exchange
- **Status**: ✓ Resolved

### Error 4: Bet Modifier Not Affecting Kill Machine
- **Issue**: Initially forgot to apply bet modifier to kill machine movement
- **Resolution**: Updated `updateKillMachine()` to calculate `actualMoveDistance = moveDistance + betModifier`
- **Status**: ✓ Resolved

### Error 5: Bless Ability Not Preventing Death
- **Issue**: Bless flag set but not checked during kill machine update
- **Resolution**: Added check in `updateKillMachine()` to detect if player would die and has Bless
- **Status**: ✓ Resolved

## Ability Card Descriptions

### Add Number Abilities
1. **2-Card**: Draw card with value 2 if available in deck
2. **3-Card**: Draw card with value 3 if available in deck
3. **4-Card**: Draw card with value 4 if available in deck
4. **6-Card**: Draw card with value 6 if available in deck
5. **7-Card**: Draw card with value 7 if available in deck

### Deck Trump Abilities
1. **Hush**: Draw a hidden card (opponent can't see value)
2. **Perfect Draw**: Draw best available card to reach target without busting
3. **Refresh**: Return all visible cards and draw 2 new ones
4. **Remove**: Remove opponent's last drawn card
5. **Return**: Return your last drawn card to deck
6. **Exchange**: Swap your last card with opponent's last card
7. **Disservice**: Force opponent to draw a card

### Bet Abilities
1. **One-Up**: Increase bet modifier by +1
2. **Two-Up**: Increase bet modifier by +2
3. **Shield**: Decrease bet modifier by -1
4. **Shield-Plus**: Decrease bet modifier by -2
5. **Bless**: Avoid death once if you would lose
6. **Bloodfeast**: Increase bet modifier by +1 AND draw an ability card
7. **Destroy**: Nullify opponent's most recent ability
8. **Friendship**: Both players draw 2 ability cards
9. **Relentless**: Destroy opponent's ability AND draw 1 ability card

### Go For Abilities
1. **Go For 17**: Change target number from 21 to 17
2. **Go For 24**: Change target number from 21 to 24
3. **Go For 27**: Change target number from 21 to 27

## AI Ability Usage Strategy by Level

### Level 1: Random
- 30% chance to use any ability randomly
- No strategic thinking
- Pure chaos factor

### Level 2: Basic
- Uses Add Number abilities when appropriate
- Uses Perfect Draw when far from target
- Simple beneficial abilities only

### Level 3: Conservative
- Prioritizes defensive abilities (Shield, Bless)
- Uses Perfect Draw safely
- Uses Return if score too high
- Survival-focused strategy

### Level 4: Smart
- Offensive: Uses Disservice, Remove when opponent ahead
- Uses bet increase abilities (One-Up, Two-Up) when winning
- Strategic Go For abilities
- Uses Destroy to nullify opponent advantages
- Adapts to game state

### Level 5: Advanced
- Urgency-based decisions (high/medium/low)
- Optimal Go For timing
- Perfect card selection with Add Number abilities
- Aggressive plays when winning
- Comeback strategies when losing
- Proactive advantage nullification
- Complex multi-factor decision making

## Integration with Existing Game

The ability card system integrates seamlessly with the existing game:

1. **Game Setup**: Each round, ability deck is shuffled and 2 cards dealt to each player
2. **Game State**: Target number and bet modifier affect gameplay dynamically
3. **Kill Machine**: Bet modifier affects machine movement distance
4. **Player Actions**: Players can use abilities through `useAbility()` method
5. **AI Integration**: AI players use `chooseAbility()` method to select abilities strategically
6. **Round Reset**: All ability-related state resets at the start of each round
